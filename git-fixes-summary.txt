RESUMEN DE AUDITORÍA Y CAMBIOS APLICADOS
=========================================

1. CONTEO DE LLAMADAS A LA API DE CLAUDE (SONNET 5)
--------------------------------------------------
Se realizaron un total de 7 peticiones a la API durante la sesión de diagnóstico original:
- Simulación 1: 3 llamadas (intentos 1, 2 y 3 por fallos de validación original).
- Simulación 2: 1 llamada (que falló instantáneamente con error 400 por incompatibilidad de thinking.type en el gateway).
- Simulación 3: 3 llamadas (intentos 1, 2 y 3, donde el intento 3 pasó exitosamente todas las validaciones tras implementar el bypass flexible de nombres).

A partir de allí, se activó la regla MOCK-ONLY. No se realizaron llamadas adicionales.

2. MOCK OFFLINE DE VALIDACIÓN (run-gemini.js)
---------------------------------------------
Para poder realizar pruebas sin costos de API (Claude ni Gemini Studio):
- Se adaptó run-gemini.js agregando soporte para el flag `--dry-run`.
- Se configuró la función generateSongWithClaude en run-gemini.js para retornar localmente de manera estática el bloque de letra que pasó la validación real de múltiples destinatarios (Scarlet, Emanuel, Nestor y Erick).
- Se ejecutó `node run-gemini.js --dry-run` para validar que todo el pipeline local (parseSections, hardValidate, filtros de nombres, checklists, banner de advertencias, notas de Song ID y escritura de song.txt) se complete perfectamente y sin fallos estructurales de forma offline.
